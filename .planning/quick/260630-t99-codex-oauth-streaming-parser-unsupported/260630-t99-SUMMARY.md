# Quick Task 260630-t99 Summary

## Outcome

Codex OAuth provider 現在可從更多 streaming/SSE event shape、wrapped JSON response 與 `status: in_progress` async response 中取出 JSON review text。若仍遇到不支援的 response shape，錯誤訊息會包含安全的結構摘要，方便下一次精準補 parser。

## Changes

- `HttpOpenAICodexOAuthReviewProvider` 支援從 nested `item`、`part`、`message`、`content` 結構擷取 `text` / `output_text`。
- 支援非 streaming JSON response 裡包一層 `response` 的 shape。
- streaming parser 會累積 event types，unsupported shape 時回報 `Event types seen: ...`。
- non-streaming JSON parser 會回報 top-level keys、type/status、output item types 與 content part types，不輸出 raw review text。
- Codex OAuth request 現在送出 `Accept: text/event-stream` 與 JSON body，避免 backend 回傳沒有 output 的 `status: in_progress` JSON response。
- 若 backend 仍回傳 `status: in_progress`，provider 會使用 response `id` 輪詢 `GET /responses/{id}`，直到 completed、terminal failure 或達到 polling limit。
- SSE parser 會讀取 `event:` 名稱作為 event type fallback，支援 data payload 省略 `type` 的 `response.output_text.delta`。
- SSE parser 會略過 pending events，若已累積完整 JSON buffer，completed/no-output event 不會觸發 retrieve。
- completed/no-output response 若沒有可用 buffer，provider 會嘗試 retrieve 完整 response；若 backend 拒絕 retrieve，錯誤會安全失敗。
- review 指令加強為只允許輸出單一 JSON object，唯一 top-level key 為 `findings`；無 finding 時必須回 `{"findings":[]}`。
- smoke command 在 JSON schema 驗證失敗時會輸出安全 top-level keys，不輸出 raw payload。
- validator 現在允許 `{"findings":[]}` 作為成功回應，因為沒有 actionable finding 是合法 review result。
- `.env.example` 新增 AI/GitHub/Codex 設定範例，包含 `CODEX_POLL_ATTEMPTS` 與 `CODEX_POLL_SLEEP_MS`。
- streaming failure/error event 會安全失敗，不會落回 API key provider。
- nested text fallback 只接受文字輸出型別，避免把 image/summary 等非 review 輸出誤當成 JSON。
- 新增 provider tests 覆蓋 nested output item event、SSE event-name fallback、wrapped response、unsupported streaming event、non-streaming shape 診斷、request headers、in-progress polling 與 polling limit。
