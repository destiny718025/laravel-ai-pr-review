# Quick Task 260630-t99: Codex OAuth Streaming Parser Unsupported Shape Fix

## Task

修正 Codex OAuth live request 在已加入 `stream => true` 後仍可能回傳 `unsupported response shape` 的問題，並讓錯誤訊息能指出實際收到的 streaming event type。

## Plan

1. 擴充 `HttpOpenAICodexOAuthReviewProvider` 的 streaming parser，支援 nested `item`、`part`、`message`、`content` 結構中的 text/output_text。
2. 支援非 streaming JSON 裡包一層 `response` 的 wrapped response shape。
3. 保留既有 `response.output_text.delta`、`response.output_text.done`、`response.completed`、`output`、`output_text` 解析路徑。
4. 對 unsupported streaming shape 收集並回報 event types，避免下一次 live 測試只能看到泛用錯誤。
5. 對 unsupported non-streaming JSON shape 回報安全的 top-level keys、type/status、output item types 與 content part types。
6. 將 Codex OAuth request 的 `Accept` header 改成 `text/event-stream`，避免 `stream => true` 仍拿到 `status: in_progress` JSON response。
7. 若 Codex backend 仍回 `status: in_progress` JSON response，使用 response `id` 輪詢 `GET /responses/{id}` 直到 completed、terminal failure 或達到上限。
8. 將輪詢次數與間隔放入 `services.codex.poll_attempts` / `services.codex.poll_sleep_ms`，並在 `.env.example` 提供範例。
9. 補上 unit tests 覆蓋 nested output item event、wrapped response、unsupported event type、non-streaming shape 診斷訊息、streaming accept header、in-progress polling 與 polling limit。
10. 執行 Codex OAuth targeted tests 與完整測試。
