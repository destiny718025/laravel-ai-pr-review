## Phase Goal

**As a** reviewer using the review run detail page, **I want to** fetch GitHub pull request data through a fakeable client boundary, **so that** later phases can reuse persisted upstream snapshots without coupling the workflow to live GitHub calls.

<objective>
Create the GitHub client contract, concrete HTTP implementation, and reusable JSON fixture boundary that all later ingestion work depends on, per D-01, D-02, D-03, D-13, D-14, and D-15.

Purpose: establish the public-only, no-token-required integration seam before the manual fetch workflow starts writing review-run data.
Output: a container-bound `GitHubClient` interface, DTOs for metadata and file snapshots, a Laravel HTTP implementation, and passing no-network tests driven by fixture files.
</objective>
