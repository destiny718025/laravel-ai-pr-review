<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub API → failure classifier | Untrusted upstream errors and exceptions cross into user-facing safe messages here. |
| failure classifier → persisted review run | Only whitelisted error copy may cross into `safe_error_message`. |
| persisted review run → detail page | The UI must render only safe stored state, never raw exception content. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-06 | Information Disclosure | `GitHubFailureMapper` / `PullRequestIngestionService` | high | mitigate | Whitelist safe error messages and keep raw GitHub responses, request headers, and token values out of persistence and view rendering. |
| T-02-07 | Denial of Service | `HttpGitHubClient` rate-limit and network handling | medium | mitigate | Detect rate-limit and transport failures explicitly and stop the workflow in a failed state instead of retrying uncontrolled in the request cycle. |
| T-02-08 | Tampering | malformed GitHub payload handling | medium | mitigate | Treat missing required keys as `malformed_response` and refuse to persist partial snapshot data. |
| T-02-SC | Tampering | dependency/tool chain | low | accept | No third-party packages are added; rely on the existing Laravel stack and containerized test commands. |
</threat_model>
