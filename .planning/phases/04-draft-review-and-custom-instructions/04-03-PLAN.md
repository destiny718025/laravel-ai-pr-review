## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-edited draft text → persisted drafts | Untrusted textarea input becomes durable review content here. |
| successful retry → preserved draft state | Fresh findings can invalidate local draft text here. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-05 | Tampering | draft state transitions | high | mitigate | Centralize edit/approve/unapprove rules in `ReviewDraftService` and reject invalid transitions server-side. |
| T-04-06 | Repudiation | retry versus draft provenance | medium | mitigate | Preserve draft rows, keep source linkage, and set `stale_at` instead of deleting rows so retry history stays auditable. |
| T-04-07 | Elevation of Privilege | approval workflow | high | mitigate | Approval remains a local status change only; no GitHub client write call, publication job, or posted transition may be added in this plan. |
| T-04-SC | Tampering | npm/pip/cargo installs | low | accept | No package installation is part of this plan. |
</threat_model>
