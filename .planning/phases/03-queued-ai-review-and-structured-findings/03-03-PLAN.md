## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app config → OpenAI adapter | Secrets cross from environment/config into the adapter and must stay out of persistence and logs. |
| OpenAI adapter → external HTTP endpoint | A future live endpoint is untrusted and must remain fully fakeable in tests. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-07 | Information Disclosure | `services.openai` secrets | high | mitigate | Keep `OPENAI_*` access in `config/services.php` only and prevent secret values from entering persisted review-run or finding data. |
| T-03-08 | Tampering | provider selection | medium | mitigate | Resolve `AIReviewProvider` through a config-gated container binding so workflow code never instantiates the concrete adapter directly. |
| T-03-09 | Denial of Service | live provider dependency in tests | medium | mitigate | Require `Http::fake()` and `Http::preventStrayRequests()` in `OpenAIReviewProviderTest`, keeping verification offline. |
| T-03-SC | Tampering | npm/pip/cargo installs | low | accept | No package-manager installs are introduced in this plan; the adapter uses the existing Laravel HTTP client. |
</threat_model>
