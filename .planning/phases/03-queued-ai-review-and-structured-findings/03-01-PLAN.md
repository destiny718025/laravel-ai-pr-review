<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → review run POST action | Untrusted user input can attempt to run review work before GitHub snapshot data exists. |
| web request → queue worker | Dispatch hands work off to an asynchronous worker that must observe committed database state only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Tampering | `POST /reviews/{reviewRun}/run` | medium | mitigate | Gate dispatch in `ReviewExecutionDispatchService::dispatch()` and reject runs without `github_fetched_at` per D-03. |
| T-03-02 | Denial of Service | `ExecuteReviewRunJob` dispatch | medium | mitigate | Use `->afterCommit()` and a queued job carrying only `reviewRunId` so duplicate inline work and stale-state workers are avoided. |
| T-03-03 | Repudiation | status transition timestamps | low | mitigate | Persist `queued_at`, `started_at`, and `completed_at` through repository methods so the audit trail is deterministic. |
| T-03-SC | Tampering | npm/pip/cargo installs | low | accept | No package-manager installs are introduced in this plan; Phase 3 uses existing Laravel queue and test tooling only. |
</threat_model>
